Capabilities

Everything you need in a personal agent

HybridClaw combines a Discord bot, a terminal UI, sandboxed tool execution, proactive scheduling, and persistent memory into a single platform.

🔒

Sandboxed Execution

Every tool runs inside an ephemeral Docker container with read-only filesystems, memory limits, and a 28+ pattern exec deny-list. Your host is never at risk.

💬

Discord & Terminal

Talk to your agent on Discord with @mentions or a custom prefix, or use the built-in TUI for local conversations. Both share the same engine.

🧠

Persistent Memory

Workspace files like SOUL.md, MEMORY.md, and IDENTITY.md give your agent personality, context, and continuity across every conversation.

⏰

Smart Scheduling

Schedule tasks with cron expressions, one-shot ISO timestamps, or simple intervals. Tasks persist in SQLite and auto-clean after execution.

💓

Proactive Heartbeat

Your agent periodically wakes up, checks its HEARTBEAT.md for tasks, and reaches out when something matters. Silent when nothing's happening.

🤖

Multi-Bot & Multi-Model

Connect to the HybridAI platform and swap between chatbots like Hermes3000 per channel. Toggle RAG for document retrieval. Each channel gets its own bot, model, and knowledge config.

🌐

Web Fetching

The agent can fetch URLs and extract readable content as markdown, with a built-in 15-minute cache. Works with HTML pages and JSON APIs.

📚

Skills System

Define reusable skills in SKILL.md files with frontmatter metadata. User-invocable via /skill-name syntax. Workspace skills override project defaults.

📊

Diagnostics & Health

HTTP health endpoint, structured Pino logging, audit trails, prompt dumps, and a status command showing uptime, sessions, and container metrics.

Architecture

Two processes, one agent

A Node.js host manages Discord, SQLite, and scheduling. An ephemeral Docker container executes tools. They talk over file-based IPC.

Container Pooling

Containers persist between requests for faster response times and auto-cleanup after a 5-minute idle timeout. Up to 5 concurrent containers.

File-Based IPC

Each session gets isolated input.json and output.json files. No sockets, no shared memory — simple, debuggable, and secure.

Resource Limits

Containers run read-only with tmpfs scratch space, 512MB memory cap, configurable CPU limits, and strict output size limits.

Get Started

Up and running in minutes

Clone, configure, build, and launch.

# Clone the repo
git clone https://github.com/bkoehler/hybridclaw.git
cd hybridclaw

# Install dependencies
npm install

# Configure environment
cp .env.example .env
# Edit .env with your Discord token and HybridAI API key

# Build the container
npm run build:container

# Start the Discord bot
npm run dev

# Or use the terminal UI
npm run tui

FAQ

Common questions

Is it safe to let the agent run shell commands?
Yes. All tools execute inside ephemeral Docker containers with read-only filesystems, memory caps, and a deny-list of 28+ dangerous command patterns. The host machine is never exposed.
Can I use it without Discord?
Absolutely. Run npm run tui to interact with your agent directly in the terminal. The TUI shares the same engine and capabilities.
What is the heartbeat system?
A configurable periodic poll (default every 30 minutes) where the agent wakes up, checks its HEARTBEAT.md file for tasks, and proactively reaches out if something needs attention. Silent acknowledgments are discarded to avoid noise.
Can different Discord channels use different bots?
Yes. Each channel can be independently configured with its own HybridAI chatbot, model, and RAG setting using the bot set and model set commands.
How does scheduling work?
Three modes: cron expressions for recurring tasks, ISO-8601 timestamps for one-shot execution, and simple intervals (minimum 10 seconds). All tasks persist in SQLite and one-shot tasks auto-delete after running.
What is HybridAI?
HybridAI is the bot platform that powers HybridClaw. It provides managed chatbots with document knowledge bases, retrieval-augmented generation (RAG), and multi-model support through an OpenAI-compatible API. You create and configure bots on the HybridAI platform, then connect them to HybridClaw.
What AI models does it support?
HybridClaw works with any model available through the HybridAI API. You can list available models with model list and switch per channel at any time.
Does the agent remember things between conversations?
Yes. Workspace files like MEMORY.md and SOUL.md persist across sessions and are injected into the system prompt on every request, giving the agent continuous context.
