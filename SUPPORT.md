## When To Open An Issue

Open a GitHub issue when you have one of these:

- a reproducible bug or regression
- a missing or broken doc page, example, or command
- a concrete setup failure with commands, versions, and logs
- a narrowly scoped feature request with a clear user problem

Use the issue forms so the report includes the version, platform, reproduction,
and validation details maintainers need.

## Security

Do not open public issues for security bugs. Follow
[SECURITY.md](./SECURITY.md) and report vulnerabilities privately to
[support@hybridai.one](mailto:support@hybridai.one?subject=HybridClaw%20security%20report).
